#### 第一步：算静态内存 (模型参数 + 梯度 + 优化器状态)

对于使用 Adam/AdamW 优化器，每个参数在训练时需要占用大约 16 Bytes 的显存

1. 模型权重 (fp16/bf16)：2 bytes

2. 梯度 (fp16/bf16)：2 bytes

3. 主权重备份 (fp32)：4 bytes

4. 优化器动量 (fp32)：4 bytes

5. 优化器方差 (fp32)：4 bytes

$$8.42 \times 10^6 \text{ parameters} \times 16 \text{ Bytes} \approx 134.7 \text{ MB}$$

#### 第二步：算动态内存 (前向传播的激活值 Activations)
假设你的配置是：Batch Size = 16, Seq Len = 512, Hidden Size = 256, Layers = 2。

在 Transformer 架构中，混合精度下的激活值显存占用有一个粗略的经验公式：

$\approx 34 \times \text{batch} \times \text{seq\_len} \times \text{hidden} \times \text{layers}$ bytes

计算： $34 \times 16 \times 512 \times 256 \times 2 \approx 142 \text{ MB}$。

加上 MoE 独有的路由（Router）计算、Top-K 索引、以及专家分配矩阵（这些中间张量往往是 fp32 格式计算的），会额外多出几十 MB。结论：激活动态内存大约需要 200 MB ~ 300 MB。

#### 隐形开销

1. CUDA Context (CUDA 上下文的“开机费”)

2. PyTorch 的显存缓存池 (Caching Allocator)

3. DDP 或 Torch Compile 的额外缓冲区 (如果你开启了的话)
